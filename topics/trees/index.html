What Is a Tree?

The One-Sentence Answer

A tree is a collection of nodes connected by edges, with one root node at the top and no cycles. Data flows downward from parent to children.

Real-World Analogy

A family tree. Your great-grandparent is the root. They have children, who have children, who have children. Everyone has exactly ONE parent (except the root). No one is their own ancestor — no loops!

Or think of a file system: / is the root, folders are nodes, files are leaves.

Types of Trees

The Tree Family

Binary Tree

Each node has at most 2 children (left and right). The most common type in interviews.

Binary Search Tree (BST)

Binary tree where left < parent < right. Enables O(log n) search.

Full Binary Tree

Every node has 0 or 2 children. Never just one child.

Complete Binary Tree

All levels full except possibly the last, which fills left to right. Used in heaps.

Perfect Binary Tree

ALL internal nodes have 2 children AND all leaves are at the same level. 2^(h+1) - 1 nodes.

Balanced Tree

Heights of left and right subtrees differ by at most 1. Guarantees O(log n) operations.

The Node — Building Block of Trees

class TreeNode {
  constructor(val) {
    this.val = val;
    this.left = null;   // left child
    this.right = null;  // right child
  }
}

// Build a simple tree:
const root = new TreeNode(10);
root.left = new TreeNode(5);
root.right = new TreeNode(15);
root.left.left = new TreeNode(3);
root.left.right = new TreeNode(7);
Memory: Unlike arrays, tree nodes are scattered in memory (like linked lists). Each node stores its value + pointers to children. No contiguous block needed.

Binary Search Tree (BST)

The Golden Rule

For every node in a BST:

  • All values in the left subtree are smaller
  • All values in the right subtree are larger
Real-World

A dictionary. Open to the middle. Is "mango" before or after "monkey"? Before → go left half. Is it before "map"? After → go right. Each step cuts the remaining pages in half!

BST Operations — Code

// ═══ SEARCH ═══
function search(node, target) {
  if (!node) return null;             // not found
  if (target === node.val) return node; // found!
  if (target < node.val)
    return search(node.left, target);   // go left
  else
    return search(node.right, target);  // go right
}
// ═══ INSERT ═══
function insert(node, val) {
  if (!node) return new TreeNode(val); // found the spot!
  if (val < node.val)
    node.left = insert(node.left, val);
  else
    node.right = insert(node.right, val);
  return node;
}
// ═══ DELETE (the tricky one!) ═══
function deleteNode(node, val) {
  if (!node) return null;
  if (val < node.val) node.left = deleteNode(node.left, val);
  else if (val > node.val) node.right = deleteNode(node.right, val);
  else {
    // Case 1: Leaf → just remove
    if (!node.left && !node.right) return null;
    // Case 2: One child → replace with child
    if (!node.left) return node.right;
    if (!node.right) return node.left;
    // Case 3: Two children → replace with inorder successor
    let successor = node.right;
    while (successor.left) successor = successor.left;
    node.val = successor.val;
    node.right = deleteNode(node.right, successor.val);
  }
  return node;
}
Delete has 3 cases: (1) Leaf: just remove. (2) One child: connect parent to grandchild. (3) Two children: replace with the inorder successor (smallest value in right subtree), then delete the successor.

BST Big O

OperationAverageWorst (unbalanced)
SearchO(log n)O(n)
InsertO(log n)O(n)
DeleteO(log n)O(n)
Find Min/MaxO(log n)O(n)
The worst case trap: If you insert sorted data [1, 2, 3, 4, 5] into a BST, it becomes a linked list! Height = n instead of log n. That's why we need balanced trees (AVL, Red-Black).

Tree Traversals — DFS (Depth-First)

The 3 DFS Orders

There are 3 ways to visit every node in depth-first order. The difference is when you process the current node:

Inorder
Left → Root → Right
Gives sorted order for BST!
Preorder
Root → Left → Right
Good for copying/serializing tree
Postorder
Left → Right → Root
Good for deleting tree

Code: All 3 Traversals

// ═══ INORDER: Left → Root → Right ═══
function inorder(node) {
  if (!node) return;
  inorder(node.left);      // 1. go left
  console.log(node.val);   // 2. process root
  inorder(node.right);     // 3. go right
}

// ═══ PREORDER: Root → Left → Right ═══
function preorder(node) {
  if (!node) return;
  console.log(node.val);   // 1. process root FIRST
  preorder(node.left);     // 2. go left
  preorder(node.right);    // 3. go right
}

// ═══ POSTORDER: Left → Right → Root ═══
function postorder(node) {
  if (!node) return;
  postorder(node.left);    // 1. go left
  postorder(node.right);   // 2. go right
  console.log(node.val);   // 3. process root LAST
}
Memory trick

"Pre" = before, "In" = in the middle, "Post" = after. The name tells you WHEN you process the root relative to the left/right subtrees. That's it!

Iterative Inorder (Using Stack)

Interviewers sometimes ask for iterative traversals. Here's the pattern with a stack:

function inorderIterative(root) {
  const result = [];
  const stack = [];
  let current = root;

  while (current || stack.length) {
    // Go as far left as possible
    while (current) {
      stack.push(current);
      current = current.left;
    }
    // Process the node
    current = stack.pop();
    result.push(current.val);
    // Move to right subtree
    current = current.right;
  }
  return result;
}
Why iterative matters: Recursive DFS uses the call stack. For very deep trees, this can cause stack overflow. Iterative version uses explicit stack — you control the memory.

BFS — Level-Order Traversal

What Is BFS on Trees?

Visit nodes level by level, from top to bottom, left to right. Uses a queue.

Real-World

Pouring water on a tree. Water flows down level by level. First the root gets wet, then its children, then grandchildren. Every node at depth d gets wet before any node at depth d+1.

Code: Level-Order Traversal

function levelOrder(root) {
  if (!root) return [];
  const result = [];
  const queue = [root];

  while (queue.length) {
    const levelSize = queue.length; // nodes at this level
    const level = [];

    for (let i = 0; i < levelSize; i++) {
      const node = queue.shift();
      level.push(node.val);
      if (node.left)  queue.push(node.left);
      if (node.right) queue.push(node.right);
    }
    result.push(level);
  }
  return result;
}
The trick: At the start of each level, queue.length tells you exactly how many nodes are on this level. Loop that many times, then you've moved to the next level.

When to Use BFS vs DFS?

Use BFS When:

Finding shortest path, level-by-level processing, minimum depth, "closest" node, right-side view, zigzag traversal

Use DFS When:

Path problems, subtree problems, validate BST, tree height, LCA, serialize/deserialize, any "bottom-up" computation

DFS Patterns — Master These!

Pattern 1: Max Depth (Height)

The most classic tree recursion. Base case + combine results from left and right.

function maxDepth(node) {
  if (!node) return 0;  // base case: empty tree has depth 0

  const leftDepth  = maxDepth(node.left);
  const rightDepth = maxDepth(node.right);

  return 1 + Math.max(leftDepth, rightDepth);
}
The recursion pattern: (1) Base case for null, (2) Recurse left, (3) Recurse right, (4) Combine and return. Almost EVERY tree problem follows this shape!

Pattern 2: Same Tree / Symmetric

// Are two trees identical?
function isSameTree(p, q) {
  if (!p && !q) return true;   // both null → same
  if (!p || !q) return false;  // one null → different
  return p.val === q.val
    && isSameTree(p.left, q.left)
    && isSameTree(p.right, q.right);
}

// Is a tree a mirror of itself?
function isSymmetric(root) {
  function mirror(a, b) {
    if (!a && !b) return true;
    if (!a || !b) return false;
    return a.val === b.val
      && mirror(a.left, b.right)   // flip!
      && mirror(a.right, b.left);  // flip!
  }
  return mirror(root.left, root.right);
}

Pattern 3: Path Sum

// Does any root-to-leaf path sum to target?
function hasPathSum(node, target) {
  if (!node) return false;

  // Leaf node? Check if remaining equals this value
  if (!node.left && !node.right)
    return node.val === target;

  // Subtract current value and check children
  return hasPathSum(node.left, target - node.val)
      || hasPathSum(node.right, target - node.val);
}

Pattern 4: Lowest Common Ancestor (LCA)

// Find the lowest common ancestor of nodes p and q
function lowestCommonAncestor(root, p, q) {
  if (!root || root === p || root === q) return root;

  const left  = lowestCommonAncestor(root.left, p, q);
  const right = lowestCommonAncestor(root.right, p, q);

  if (left && right) return root;  // p and q are on different sides
  return left || right;            // both on same side
}
Intuition

Two people climbing from different leaves toward the root. The first node where their paths cross is the LCA. The algorithm checks: are p and q on different sides? If yes, current node IS the LCA.

Pattern 5: Validate BST

function isValidBST(node, min = -Infinity, max = Infinity) {
  if (!node) return true;

  if (node.val <= min || node.val >= max) return false;

  return isValidBST(node.left, min, node.val)   // left must be < node
      && isValidBST(node.right, node.val, max);  // right must be > node
}
Common mistake: Only checking left.val < node.val is NOT enough! You must check that ALL values in the left subtree are smaller, not just the direct child. Pass the range (min, max) down.

Pattern 6: Invert a Binary Tree

function invertTree(node) {
  if (!node) return null;

  // Swap left and right
  [node.left, node.right] = [node.right, node.left];

  invertTree(node.left);
  invertTree(node.right);

  return node;
}
Fun fact: This is the problem that became a meme. It's 4 lines of code, but it teaches you how tree recursion works. Swap at each node, recurse on both children.

Pattern 7: Diameter of Binary Tree

function diameterOfBinaryTree(root) {
  let diameter = 0;

  function height(node) {
    if (!node) return 0;
    const left = height(node.left);
    const right = height(node.right);

    // Diameter through this node = left + right
    diameter = Math.max(diameter, left + right);

    return 1 + Math.max(left, right);
  }

  height(root);
  return diameter;
}
Key insight: The diameter (longest path between any two nodes) doesn't have to pass through the root! At each node, the path through it = left height + right height. Track the global max.

Balanced Trees — Why They Matter

The Problem with Unbalanced BSTs

Balanced vs Unbalanced
Balanced: O(log n)
Unbalanced: O(n)

Inserting [1, 2, 3, 4, 5] into a BST gives you a straight line (linked list). Every operation becomes O(n). Balanced trees fix this.

Check if a Tree is Balanced

function isBalanced(node) {
  function check(node) {
    if (!node) return 0;
    const left = check(node.left);
    if (left === -1) return -1;
    const right = check(node.right);
    if (right === -1) return -1;

    if (Math.abs(left - right) > 1) return -1; // unbalanced!
    return 1 + Math.max(left, right);
  }
  return check(node) !== -1;
}
Trick: Return -1 as a "flag" when unbalanced. This avoids computing the full height when we already know the answer. Early termination!

Self-Balancing Trees (Know These Exist)

AVL Tree

Strictly balanced — heights differ by at most 1. Uses rotations after insert/delete. Faster lookup than Red-Black.

Red-Black Tree

Loosely balanced — guaranteed O(log n) but less rotations than AVL. Used in Java TreeMap, C++ std::map.

B-Tree / B+ Tree

Multi-way tree (more than 2 children). Used in databases and file systems. Optimized for disk I/O.

Trie (Prefix Tree)

Each node represents a character. Used for autocomplete, spell check, IP routing. Separate deep dive!

Interview reality: You won't be asked to implement AVL or Red-Black from scratch. But you MUST know they exist, why they're needed, and their O(log n) guarantees.

Heaps — Priority Queues

What Is a Heap?

A heap is a complete binary tree where every parent is smaller (min-heap) or larger (max-heap) than its children.

Real-World

An emergency room queue. The most critical patient gets treated first, regardless of arrival order. A heap always gives you the min (or max) in O(1).

Heap as an Array

The brilliant trick: a complete binary tree can be stored in an array (no pointers needed!):

Heap → Array mapping
// For node at index i:
parent(i)     = Math.floor((i - 1) / 2)
leftChild(i)  = 2 * i + 1
rightChild(i) = 2 * i + 2
No pointers! Because heaps are complete binary trees, the array never has gaps. Parent-child relationships are pure math, just like array index calculations.

Essential Tree Tricks

Trick 1: Serialize & Deserialize a Tree

// Convert tree → string (preorder with nulls)
function serialize(root) {
  if (!root) return 'null';
  return root.val + ',' + serialize(root.left)
                  + ',' + serialize(root.right);
}

// Convert string → tree
function deserialize(str) {
  const vals = str.split(',');
  let i = 0;
  function build() {
    if (vals[i] === 'null') { i++; return null; }
    const node = new TreeNode(+vals[i++]);
    node.left = build();
    node.right = build();
    return node;
  }
  return build();
}

Trick 2: Build Tree from Traversals

// Build tree from preorder + inorder
function buildTree(preorder, inorder) {
  if (!preorder.length) return null;

  const rootVal = preorder[0];
  const root = new TreeNode(rootVal);
  const mid = inorder.indexOf(rootVal);

  root.left = buildTree(
    preorder.slice(1, mid + 1),   // left part of preorder
    inorder.slice(0, mid)          // left part of inorder
  );
  root.right = buildTree(
    preorder.slice(mid + 1),       // right part of preorder
    inorder.slice(mid + 1)         // right part of inorder
  );
  return root;
}
Key insight: Preorder's first element is always the root. Find that root in inorder — everything left of it is the left subtree, everything right is the right subtree. Recurse!

Trick 3: Morris Traversal — O(1) Space

Traverse a tree without recursion AND without a stack. Uses temporary thread pointers:

function morrisInorder(root) {
  const result = [];
  let current = root;

  while (current) {
    if (!current.left) {
      result.push(current.val);  // visit
      current = current.right;
    } else {
      // Find inorder predecessor
      let pred = current.left;
      while (pred.right && pred.right !== current)
        pred = pred.right;

      if (!pred.right) {
        pred.right = current;    // create thread
        current = current.left;
      } else {
        pred.right = null;       // remove thread
        result.push(current.val); // visit
        current = current.right;
      }
    }
  }
  return result;
}
Advanced technique. O(n) time, O(1) space (ignoring output). Temporarily modifies the tree but restores it. Asked in hard interviews — know it exists.
