What Is a String?

The One-Sentence Answer

A string is a sequence of characters stored next to each other in memory. Just like an array, but each element is a character (letter, digit, symbol).

Real-World Analogy

A word on paper. "HELLO" is 5 characters sitting side by side. H-E-L-L-O. You can grab any character by position: index 0 is 'H', index 3 is 'L'.

Strings and arrays are nearly identical — except strings have meaning (they spell words) and most languages treat them specially (immutability, methods, etc.).

String Fundamentals

  • Length: Count of characters (O(1) in most languages)
  • Index access: Get char at position — O(1)
  • Immutable in Java, Python, JavaScript: Once created, can't change. (More on this next!)
  • Mutable in C++, Go: Can modify characters in place
  • Null-terminated vs length-prefixed: C uses null '\0' at end; Python stores length separately
Interview fact: "String" problems are often "array problems in disguise" — use 2-pointers, sliding window, frequency arrays, etc.
⚠️

Immutability — The #1 Trap

The Problem: String += in a Loop is O(n²)!

In Java, Python, JavaScript: strings are immutable. When you do s += char, the language creates a NEW string and copies all old characters into it. Then discards the old one.

Wrong (O(n²)):
let result = "";
for (let char of word) { result += char; } // Creates n strings!
Right (O(n)):
let chars = [];
for (let char of word) { chars.push(char); }
return chars.join(""); // One allocation at the end

The Takeaway

  • Use StringBuilder (Java), list (Python), array + join (JS): Build in O(n), not O(n²)
  • Immutability is a feature, not a bug: Makes strings safe in multithreading
  • For interviews: Always use StringBuilder-like patterns when building strings

Operations & Big O Complexity

String Operation Costs

Operation Time Space Note
Access by index: s[i] O(1) O(1) Direct memory access
Length: len(s) O(1) O(1) Stored in header
Substring: s[i:j] O(j-i) O(j-i) Must copy chars
Concatenation: s1 + s2 O(n+m) O(n+m) Copy both strings

Key Insight

Most string operations scan the entire string = O(n). This is why they're easier than graph problems but harder than array access.

The tricks in interviews are to avoid rescanning the string multiple times. Use sliding window, two pointers, frequency counting — scan once or twice, not three times!

Palindromes — Expand Around Center

The Golden Approach: O(n²) Time, O(1) Space

A palindrome reads the same forwards and backwards: "racecar", "madam", "noon".

Trick: For each possible center, expand outward until chars don't match. Try even-length (center between two chars) and odd-length (center on a char).

Real-World

Think of a mirror in the middle. Match pairs on both sides. "r|acecar" — the center is between 'a' and 'c', and they don't match, so there's no 2-char palindrome here.

Code: Longest Palindromic Substring

function longestPalindrome(s) {
  let maxLen = 0, start = 0;

  function expandAroundCenter(left, right) {
    while (left >= 0 && right < s.length && s[left] === s[right]) {
      left--; right++;
    }
    return right - left - 1; // length of palindrome
  }

  for (let i = 0; i < s.length; i++) {
    // Odd-length palindrome (center on char)
    let len1 = expandAroundCenter(i, i);
    // Even-length palindrome (center between chars)
    let len2 = expandAroundCenter(i, i + 1);

    let len = Math.max(len1, len2);
    if (len > maxLen) {
      maxLen = len;
      start = i - Math.floor((len - 1) / 2);
    }
  }
  return s.substring(start, start + maxLen);
}

Anagrams & Frequency Counting

The Definition

Two strings are anagrams if they contain the same characters in different orders. "listen" and "silent" are anagrams.

The trick: Count character frequencies. If both strings have the same frequency map, they're anagrams.

Real-World

You have a bag of Scrabble tiles. Can you spell "silent" with the tiles from "listen"? Only if the bags have identical tiles.

3 Approaches

Approach 1: Frequency Array (Best for ASCII)

O(n) time, O(1) space — Use an array of size 26 (for 'a'-'z') or 256 (ASCII). Count both strings, compare.

Approach 2: HashMap (Best for Unicode)

O(n) time, O(k) space (k=unique chars) — Handle any language, not just 26 letters.

Approach 3: Sort & Compare (Simple but O(n log n))

Sort both strings. If sorted versions are equal, they're anagrams. Easy to code, slower.

Sliding Window on Strings

The Pattern: "Longest Substring Without Repeating Characters"

Find the longest substring with no duplicate characters. Example: "abcabcbb" → "abc" (length 3).

Approach: Use two pointers (left, right) forming a window. Move right to expand, move left to shrink when a duplicate is found. Track chars in a Set or frequency map.

function lengthOfLongestSubstring(s) {
  let charIndex = {}; // last seen index of each char
  let maxLen = 0, left = 0;

  for (let right = 0; right < s.length; right++) {
    if (charIndex[s[right]] !== undefined && charIndex[s[right]] >= left) {
      // Duplicate found, shrink from left
      left = charIndex[s[right]] + 1;
    }
    charIndex[s[right]] = right;
    maxLen = Math.max(maxLen, right - left + 1);
  }
  return maxLen;
}

Two Pointers on Strings

The Classic Problems

  • Valid Palindrome: Ignore non-alphanumeric, use left & right pointers
  • Reverse String: Two pointers start at ends, swap and move inward
  • Reverse Words: Reverse entire string, then reverse each word
  • Container With Most Water: Start at ends, move the shorter pointer inward

The Pattern: Why It Works

Two pointers eliminate redundant checks. Instead of comparing every char to every other char (O(n²)), you compare pairs moving toward the center (O(n)).

Whenever you see "palindrome", "valid", or "reverse" in a string problem, think two pointers first.

StringBuilder / Buffer Pattern

When & Why to Use It

Use StringBuilder (or array + join) when building a string character by character.

  • Java: StringBuilder sb = new StringBuilder(); then sb.append(char)
  • Python: chars = []; chars.append(char) then ''.join(chars)
  • JavaScript: let arr = []; arr.push(char) then arr.join('')
  • Go: var buf bytes.Buffer; buf.WriteRune(ch)

Why? String concatenation is O(n), done n times = O(n²). StringBuilder amortizes to O(n).

Code Comparison

// ❌ SLOW (O(n²))
let result = "";
for (let i = 0; i < n; i++) {
  result += someChar; // New string created each time
}

// ✅ FAST (O(n))
let result = [];
for (let i = 0; i < n; i++) {
  result.push(someChar); // O(1) amortized
}
return result.join(''); // O(n) final join

Common Use Cases

Reverse a String

Build characters in reverse order into StringBuilder.

Run-Length Encoding

Count consecutive chars, append count + char to builder.

Remove Duplicates / Whitespace

Iterate through, selectively append to builder.

Pattern Matching: Naive & KMP

The Problem: Find All Occurrences of a Substring

Find the pattern "AB" in text "ABABAB". Return all start indices: [0, 2, 4].

Naive approach: For each position in text, check if pattern matches. O(n×m) in worst case.

KMP Algorithm (Brief)

KMP = Knuth-Morris-Pratt. Idea: Build a "failure function" that tells you how much to skip when a mismatch occurs.

Time: O(n + m) instead of O(n×m). Only matters for large m or many mismatches.

Rarely asked in interviews, but good to know the idea: failure function tracks overlap in the pattern itself.

For interviews, naive search is usually fine. Mention KMP if time is critical, but know you might not have time to code it perfectly.

Essential Tricks & Techniques

Trick 1: String Rotation Check — (s+s).includes(t)

Check if string `t` is a rotation of string `s`. Example: "waterbottle" contains "bottlewater".

function isRotation(s, t) {
  if (s.length !== t.length) return false;
  return (s + s).includes(t);
}

Why it works: All rotations of "abc" appear in "abcabc": "abc", "bca", "cab".

Trick 2: Character Index — c - 'a' for 0-25 indexing

Convert char 'a' to 0, 'b' to 1, ..., 'z' to 25.

// Build frequency array for lowercase letters
let freq = new Array(26).fill(0);
for (let char of word) {
  freq[char.charCodeAt(0) - 'a'.charCodeAt(0)]++;
}

Trick 3: StringBuilder for Reverse

Don't use string concatenation. Use array + reverse + join.

function reverse(s) {
  return s.split('').reverse().join('');
  // In-place in languages with mutable strings
}

Trick 4: XOR for Single Difference

Find the one character that differs between two similar strings.

// If strings differ by exactly one char
let diff = 0;
for (let i = 0; i < s.length; i++) {
  diff ^= s.charCodeAt(i) ^ t.charCodeAt(i);
}
return String.fromCharCode(diff);

Trick 5: Comparing Char Arrays Instead of Strings

For anagrams or character-order problems, compare sorted char arrays — often cleaner than string manipulation.

function areAnagrams(s, t) {
  return s.split('').sort().join('') === t.split('').sort().join('');
}

Trade-off: Easier to code, O(n log n) instead of O(n), but often acceptable.
