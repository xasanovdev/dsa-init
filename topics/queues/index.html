What Is a Queue?

The FIFO Principle

A queue is a linear data structure that follows FIFO — First In, First Out. Think of any real-life line: the person who joins first gets served first.

  • Enqueue — add an item to the rear (back) of the queue
  • Dequeue — remove an item from the front of the queue
  • Peek / Front — look at the front item without removing it
Real-world analogy: A queue is like a line at a coffee shop. You join at the back, wait your turn, and the barista serves whoever is at the front first. No cutting!

Under the Hood

Array-Based Queue

The simplest approach uses an array with two pointers: front and rear. But naive array queues waste space — after dequeuing, the front slots go unused. That's why we use a circular array.

class ArrayQueue {
  constructor(capacity) {
    this.data = new Array(capacity);
    this.front = 0;
    this.rear = 0;
    this.size = 0;
    this.capacity = capacity;
  }
  enqueue(val) {
    if (this.size === this.capacity) return;
    this.data[this.rear] = val;
    this.rear = (this.rear + 1) % this.capacity; // wrap around!
    this.size++;
  }
  dequeue() {
    if (this.size === 0) return undefined;
    const val = this.data[this.front];
    this.front = (this.front + 1) % this.capacity;
    this.size--;
    return val;
  }
}

Linked-List-Based Queue

A linked list is actually a perfect fit for queues. Maintain a head pointer (front) and tail pointer (rear). Enqueue at tail, dequeue at head — both O(1) with no wasted space.

class ListQueue {
  constructor() {
    this.head = null;
    this.tail = null;
    this.size = 0;
  }
  enqueue(val) {
    const node = { val, next: null };
    if (this.tail) this.tail.next = node;
    this.tail = node;
    if (!this.head) this.head = node;
    this.size++;
  }
  dequeue() {
    if (!this.head) return undefined;
    const val = this.head.val;
    this.head = this.head.next;
    if (!this.head) this.tail = null;
    this.size--;
    return val;
  }
}
In JS interviews: Just use an array with push() to enqueue and shift() to dequeue. It works but shift() is O(n). For real performance, use a linked list or circular buffer. Interviewers typically accept the simple array approach.

Types of Queues

1. Simple Queue (Regular Queue)

Standard FIFO — enqueue at rear, dequeue at front. What we've been discussing.

2. Circular Queue

The rear wraps around to the front when it reaches the end of the array. Prevents wasted space. Used in buffering (like streaming audio/video), CPU scheduling, and traffic systems.

// The key insight: modulo arithmetic
this.rear = (this.rear + 1) % this.capacity;
this.front = (this.front + 1) % this.capacity;

3. Deque (Double-Ended Queue)

You can add and remove from both ends. This is the Swiss Army knife of queues. Python's collections.deque and JavaScript's approach with arrays give you this power.

  • addFront / addRear — insert at either end
  • removeFront / removeRear — remove from either end

4. Priority Queue

Items have priorities. The highest-priority item gets dequeued first, regardless of insertion order. Usually implemented with a heap (not a regular queue). We'll cover this in detail below.

Analogy for each type: Simple queue = coffee shop line. Circular queue = merry-go-round — when the last seat fills, the next person waits for the first seat to open. Deque = a card deck where you can draw from top or bottom. Priority queue = hospital ER — the sickest patient is treated first regardless of arrival time.

BFS — Queues in Action

Breadth-First Search

BFS is the #1 reason queues matter in DSA. BFS explores nodes level by level — it visits all neighbors before going deeper. The queue ensures we process nodes in the order we discover them.

function bfs(root) {
  if (!root) return [];
  const result = [];
  const queue = [root]; // start with root

  while (queue.length > 0) {
    const node = queue.shift(); // dequeue from front
    result.push(node.val);

    if (node.left) queue.push(node.left);  // enqueue children
    if (node.right) queue.push(node.right);
  }
  return result;
}

BFS Level-Order Template

When you need to process level by level (like finding the depth, or zigzag traversal), track the level size:

function levelOrder(root) {
  if (!root) return [];
  const result = [], queue = [root];

  while (queue.length) {
    const levelSize = queue.length; // snapshot!
    const level = [];
    for (let i = 0; i < levelSize; i++) {
      const node = queue.shift();
      level.push(node.val);
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }
    result.push(level);
  }
  return result; // [[1], [2,3], [4,5,6,7]]
}
BFS golden rule: If a problem says "shortest path in an unweighted graph" or "minimum moves" — think BFS. BFS always finds the shortest path first because it explores level by level.

Deque — Double-Ended Queue

Why Deques Matter

A deque lets you push/pop from both front and back in O(1). This makes it perfect for problems where you need to maintain a window of elements and efficiently remove from either end.

  • Sliding Window Maximum — the classic deque problem
  • Palindrome checker — compare from both ends
  • Work-stealing algorithms — threads steal from the back of other queues

Sliding Window Maximum

The Problem

Given an array and a window size k, find the maximum element in every window of size k as it slides from left to right. This is THE classic deque interview problem.

Brute force: O(n×k). With a monotonic deque: O(n).

The Trick: Monotonic Decreasing Deque

Maintain a deque of indices where the values are in decreasing order. The front of the deque always holds the index of the current window's maximum.

  • Before adding a new element, remove all smaller elements from the rear
  • Remove elements from the front if they're outside the window
  • The front always gives you the maximum
function maxSlidingWindow(nums, k) {
  const deque = []; // stores indices
  const result = [];

  for (let i = 0; i < nums.length; i++) {
    // Remove indices outside the window
    while (deque.length && deque[0] <= i - k)
      deque.shift();

    // Remove smaller values from rear
    while (deque.length && nums[deque[deque.length-1]] <= nums[i])
      deque.pop();

    deque.push(i);

    if (i >= k - 1)
      result.push(nums[deque[0]]); // front = max
  }
  return result;
}

Priority Queue

What Is a Priority Queue?

A priority queue is an abstract data type where each element has a priority. Instead of FIFO, the element with the highest priority is dequeued first. It's typically implemented using a binary heap.

When to Use Priority Queues

  • Kth Largest/Smallest — maintain a min-heap of size k
  • Merge K Sorted Lists — push heads of all lists into min-heap
  • Dijkstra's Algorithm — always process the nearest unvisited node
  • Task Scheduling — process highest priority tasks first
  • Median in a Stream — use two heaps (max-heap + min-heap)

Min Heap Implementation (Simplified)

class MinHeap {
  constructor() { this.heap = []; }
  push(val) {
    this.heap.push(val);
    this._bubbleUp(this.heap.length - 1);
  }
  pop() {
    const min = this.heap[0];
    const last = this.heap.pop();
    if (this.heap.length) {
      this.heap[0] = last;
      this._sinkDown(0);
    }
    return min;
  }
  _bubbleUp(i) {
    while (i > 0) {
      const parent = (i - 1) >> 1;
      if (this.heap[parent] <= this.heap[i]) break;
      [this.heap[parent], this.heap[i]] =
        [this.heap[i], this.heap[parent]];
      i = parent;
    }
  }
  _sinkDown(i) {
    const n = this.heap.length;
    while (true) {
      let smallest = i;
      const l = 2 * i + 1, r = 2 * i + 2;
      if (l < n && this.heap[l] < this.heap[smallest]) smallest = l;
      if (r < n && this.heap[r] < this.heap[smallest]) smallest = r;
      if (smallest === i) break;
      [this.heap[smallest], this.heap[i]] =
        [this.heap[i], this.heap[smallest]];
      i = smallest;
    }
  }
}
Analogy: A priority queue is like an ER triage system. A patient with a broken arm (priority 5) might arrive before a heart attack patient (priority 1), but the heart attack gets treated first. The "queue" isn't really a line — it's sorted by urgency.

Classic Design Problems

Implement Queue Using Two Stacks

A classic interview question! Use two stacks: one for enqueue (inStack), one for dequeue (outStack). When dequeue is called and outStack is empty, pour all elements from inStack into outStack — this reverses the order, giving us FIFO behavior.

class MyQueue {
  constructor() {
    this.inStack = [];  // for push
    this.outStack = []; // for pop
  }
  push(x) {
    this.inStack.push(x);
  }
  pop() {
    this._transfer();
    return this.outStack.pop();
  }
  peek() {
    this._transfer();
    return this.outStack[this.outStack.length - 1];
  }
  _transfer() {
    if (!this.outStack.length) {
      while (this.inStack.length)
        this.outStack.push(this.inStack.pop());
    }
  }
  empty() {
    return !this.inStack.length && !this.outStack.length;
  }
}
Amortized O(1): Each element is moved from inStack to outStack at most once. So even though a single pop might be O(n), across n operations each element is touched only twice total → amortized O(1) per operation.

Implement Stack Using Two Queues

The reverse problem! Push is expensive: after adding to q2, move everything from q1 to q2, then swap q1 and q2.

class MyStack {
  constructor() {
    this.q1 = [];
    this.q2 = [];
  }
  push(x) {
    this.q2.push(x);
    while (this.q1.length)
      this.q2.push(this.q1.shift());
    [this.q1, this.q2] = [this.q2, this.q1];
  }
  pop()  { return this.q1.shift(); }
  top()  { return this.q1[0]; }
  empty(){ return !this.q1.length; }
}

Design Circular Queue (LeetCode 622)

Implement a fixed-size circular queue with: enQueue, deQueue, Front, Rear, isEmpty, isFull. The key insight: use modulo arithmetic to wrap the rear pointer around.

class MyCircularQueue {
  constructor(k) {
    this.data = new Array(k);
    this.head = 0; this.count = 0; this.cap = k;
  }
  enQueue(val) {
    if (this.isFull()) return false;
    this.data[(this.head + this.count) % this.cap] = val;
    this.count++;
    return true;
  }
  deQueue() {
    if (this.isEmpty()) return false;
    this.head = (this.head + 1) % this.cap;
    this.count--;
    return true;
  }
  Front()   { return this.isEmpty() ? -1 : this.data[this.head]; }
  Rear()    { return this.isEmpty() ? -1 : this.data[(this.head+this.count-1)%this.cap]; }
  isEmpty() { return this.count === 0; }
  isFull()  { return this.count === this.cap; }
}

Queue Tricks & Patterns

1. Multi-Source BFS

Instead of starting BFS from one node, start from multiple sources at once. Push all starting nodes into the queue initially. Used in "Rotting Oranges", "Walls and Gates", "01 Matrix".

// Push ALL rotten oranges into queue first
for (let r = 0; r < rows; r++)
  for (let c = 0; c < cols; c++)
    if (grid[r][c] === 2) queue.push([r, c]);
// Then BFS spreads from all sources simultaneously

2. BFS Grid Template

The 4-directional BFS pattern you'll use constantly for grid problems:

const dirs = [[0,1],[0,-1],[1,0],[-1,0]];
const queue = [[startR, startC]];
visited[startR][startC] = true;

while (queue.length) {
  const [r, c] = queue.shift();
  for (const [dr, dc] of dirs) {
    const nr = r + dr, nc = c + dc;
    if (nr >= 0 && nr < rows && nc >= 0 && nc < cols
        && !visited[nr][nc]) {
      visited[nr][nc] = true;
      queue.push([nr, nc]);
    }
  }
}

3. Level Tracking

When you need to know which "level" or "step" you're at (like counting BFS layers), use the snapshot technique: save queue.length before processing the level.

4. Deque for Monotonic Problems

Whenever you see "maximum/minimum in a sliding window" or "first greater/smaller in a range", think monotonic deque. Keep the deque sorted by removing elements that break the monotonic property.

5. Task Scheduling Pattern

Many scheduling problems use a queue + cooldown approach. Process tasks, put them in a cooldown queue, and re-add them when they're ready.

// Task Scheduler (LeetCode 621) pattern
// Use a max-heap for highest-freq tasks
// Use a queue for cooling-down tasks: [count, readyTime]
const cooldown = []; // queue of [remaining, readyAt]
while (heap.size || cooldown.length) {
  time++;
  if (heap.size) {
    const count = heap.pop() - 1;
    if (count) cooldown.push([count, time + n]);
  }
  if (cooldown.length && cooldown[0][1] === time)
    heap.push(cooldown.shift()[0]);
}
Pattern cheat sheet: See "shortest path"? → BFS. See "level by level"? → BFS with level tracking. See "sliding window max/min"? → Monotonic deque. See "implement X using Y"? → Two stacks ↔ queue trick. See "schedule tasks"? → Queue + heap.

Quiz — Test Yourself

1. What does FIFO stand for and how is it different from LIFO?
FIFO = First In, First Out. The first element added is the first one removed (like a line). LIFO = Last In, First Out (a stack). The key difference: a queue removes from the front, a stack removes from the top.
2. What is the time complexity of enqueue, dequeue, and peek?
All three are O(1). Enqueue adds to the rear, dequeue removes from the front, peek looks at the front — all constant time with proper implementation.
3. Why is a circular queue better than a simple array queue?
In a simple array queue, dequeued slots become wasted space. A circular queue reuses those slots by wrapping the rear pointer around using modulo arithmetic: rear = (rear + 1) % capacity. This gives O(1) operations without wasting memory.
4. How does "Implement Queue using Two Stacks" achieve amortized O(1)?
Each element is pushed once to inStack and popped+pushed once to outStack. That's 3 operations per element total. Across n operations, the total work is O(n), giving amortized O(1) per operation. The transfer from inStack to outStack only happens when outStack is empty.
5. Why is BFS guaranteed to find the shortest path in an unweighted graph?
BFS explores nodes level by level. It processes all nodes at distance d before any node at distance d+1. So the first time BFS reaches a node, it has found the shortest path to that node. The queue ensures this level-by-level ordering.
6. What is a monotonic deque and when would you use it?
A monotonic deque is a deque where elements are maintained in sorted order (either increasing or decreasing). You use it for sliding window max/min problems. Before adding a new element, remove all elements from the rear that violate the monotonic property. The front always holds the current window's extreme value.
7. In the Sliding Window Maximum problem, why do we store indices instead of values in the deque?
We need to know when an element leaves the window. By storing indices, we can check if the front element's index is outside the current window (i.e., index ≤ i - k) and remove it. If we only stored values, we couldn't tell which ones are expired.
8. What is multi-source BFS?
Multi-source BFS starts from multiple nodes simultaneously by pushing all source nodes into the queue before starting. It's used when you need to find distances from any of several sources (like "Rotting Oranges" where all rotten oranges spread simultaneously). Same BFS, just multiple starting points.
9. What's the time complexity of a priority queue's insert and extract operations?
Both insert and extract-min/max are O(log n) because they need to maintain the heap property by bubbling up or sinking down through a binary heap of height log n. Peeking at the min/max is O(1) since it's always at the root.
10. When you see "shortest path" or "minimum steps" in a problem, what should you think of?
BFS! In an unweighted graph (or grid where each step costs 1), BFS finds the shortest path. Use a queue, process level by level, and the first time you reach the target is the shortest path. If edges have different weights, use Dijkstra's (priority queue) instead.
