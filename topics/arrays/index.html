What Is an Array?

The One-Sentence Answer

An array is a list of items stored next to each other in memory. That's it. Items sit side by side like books on a shelf.

Real-World Analogy

A row of lockers in school. Locker #0, Locker #1, Locker #2... Each one holds something. You know exactly where each locker is because they're numbered and side by side.

Want locker #5? Walk straight to it. Don't need to check lockers #0–4.

Under the Hood — Memory Layout

How the Computer Actually Stores an Array

When you write let arr = [10, 20, 30, 40, 50], the computer reserves a block of consecutive memory addresses:

Each number takes 4 bytes. Address of arr[i] = start + (i × 4)

This is the magic formula! The computer doesn't "search" for index 3. It calculates the address directly:

address of arr[3] = 0x100 + (3 × 4) = 0x10C → value is 40
// That's why array access is O(1) — it's just MATH!

Why "Contiguous" Matters

The Key Insight

Contiguous = items are stored RIGHT NEXT TO EACH OTHER. No gaps.

This is WHY random access is O(1). The CPU knows exactly where to look because it can do simple math.

Compare with a Linked List where items can be scattered everywhere in memory — you have to follow pointers one by one. That's O(n).

Static vs Dynamic Arrays

The Two Types

Static Array

Fixed size. You declare the size upfront and it can't change.

int arr[5]; // C/C++ — always 5 slots
int[] arr = new int[5]; // Java

Used in: C, C++, Java (primitive arrays)

Dynamic Array

Auto-resizes. It grows when you add items. Starts small, doubles when full.

arr = []  # Python list
let arr = []; // JavaScript
ArrayList<Integer> // Java

Used in: Python, JavaScript, Java (ArrayList)

Pattern: Two Pointers

What Is It?

Use two variables (pointers) that move through the array. They can start from both ends and walk toward each other, or both from the start at different speeds.

Real-World

Two people reading the same book. One starts from page 1, the other from the last page. They walk toward each other until they meet in the middle.

This turns O(n²) brute force into O(n).

Code: Two Sum (Sorted Array)

function twoSum(arr, target) {
  let left = 0;
  let right = arr.length - 1;

  while (left < right) {
    const sum = arr[left] + arr[right];

    if (sum === target) return [left, right];
    if (sum < target) left++;   // need bigger → move left up
    else              right--;  // need smaller → move right down
  }
  return [-1, -1]; // not found
}
Why it works: Array is sorted. If sum is too small, we move left pointer right (bigger number). If too big, we move right pointer left (smaller number). We shrink the window until we find the answer.

Classic Two Pointer Problems

Reverse an Array

Left starts at 0, right at end. Swap them. Move inward. O(n) time, O(1) space.

Remove Duplicates (Sorted)

Slow pointer marks where to write. Fast pointer scans ahead. Skip duplicates.

Container With Most Water

Two pointers from edges. Move the shorter side inward to try for more area.

3Sum (Three Sum)

Sort array. Fix one number, then use two pointers on the rest. O(n²).

Pattern: Sliding Window

What Is It?

A "window" (subarray) that slides across the array. Instead of recalculating everything from scratch, you add the new element and remove the old one.

Real-World

Looking through a train window. The window is fixed size. As the train moves, new scenery enters from one side and old scenery leaves from the other. You don't need to re-see the whole journey to know what's currently visible.

Turns O(n × k) brute force into O(n).

Code: Fixed Window — Max Sum Subarray

function maxSubarraySum(arr, k) {
  let windowSum = 0;
  // Build the first window
  for (let i = 0; i < k; i++) windowSum += arr[i];

  let maxSum = windowSum;

  // Slide: add new right, remove old left
  for (let i = k; i < arr.length; i++) {
    windowSum += arr[i] - arr[i - k];  // THE TRICK!
    maxSum = Math.max(maxSum, windowSum);
  }
  return maxSum;
}
The key line: windowSum += arr[i] - arr[i - k] — Add the new element entering the window, subtract the one leaving. No need to re-sum the entire window!

Variable-Length Sliding Window

Sometimes the window size isn't fixed. You expand the right side and shrink the left side based on a condition:

// Smallest subarray with sum >= target
function minSubarrayLen(target, arr) {
  let left = 0, sum = 0, minLen = Infinity;

  for (let right = 0; right < arr.length; right++) {
    sum += arr[right];           // expand window

    while (sum >= target) {      // shrink window
      minLen = Math.min(minLen, right - left + 1);
      sum -= arr[left];           // remove left
      left++;                     // shrink
    }
  }
  return minLen === Infinity ? 0 : minLen;
}

When to Use Sliding Window?

Look for keywords: "subarray", "substring", "consecutive", "contiguous", "window of size k", "minimum/maximum subarray".

Pattern: Prefix Sum

What Is It?

Pre-compute a running total so you can find the sum of ANY range in O(1).

Real-World

Running total of your bank account. If you know you had $500 after Monday and $800 after Wednesday, then you earned $800 - $500 = $300 on Tue+Wed. You didn't need to re-add each transaction!

The Formula

// Build prefix sum: prefix[i] = sum of arr[0..i]
prefix[0] = arr[0];
for (let i = 1; i < n; i++)
  prefix[i] = prefix[i-1] + arr[i];

// Query: sum from index L to R
sum(L, R) = prefix[R] - prefix[L-1]
// (if L === 0, just use prefix[R])
Power move: Build prefix sum in O(n), then answer UNLIMITED range queries in O(1) each. Without it, each query is O(n).

The Brilliant Idea

At each position, ask: "Is it better to extend the current subarray, or start fresh from here?"

function maxSubarraySum(arr) {
  let currentSum = arr[0];
  let maxSum = arr[0];

  for (let i = 1; i < arr.length; i++) {
    // KEY DECISION: extend or start fresh?
    currentSum = Math.max(arr[i], currentSum + arr[i]);
    maxSum = Math.max(maxSum, currentSum);
  }
  return maxSum;
}
The trick: If currentSum goes negative, it means the past is dragging you down. Start fresh! Math.max(arr[i], currentSum + arr[i]) handles this automatically.

Pattern: Dutch National Flag

What Is It?

Sort an array with only 3 distinct values in one pass. Uses three pointers: low, mid, high.

Real-World

Sorting colored balls into 3 buckets — red, white, blue. You scan through, and swap each ball into its correct zone.

Classic problem: Sort Colors (LeetCode #75) — sort array of 0s, 1s, and 2s in one pass, O(1) space.

function sortColors(arr) {
  let low = 0, mid = 0, high = arr.length - 1;

  while (mid <= high) {
    if (arr[mid] === 0) {
      [arr[low], arr[mid]] = [arr[mid], arr[low]];
      low++; mid++;
    } else if (arr[mid] === 1) {
      mid++;  // already in right place
    } else { // arr[mid] === 2
      [arr[mid], arr[high]] = [arr[high], arr[mid]];
      high--; // don't advance mid — need to check swapped value
    }
  }
}

Binary Search on Arrays

The Golden Rule

If the array is sorted, you can use binary search to find things in O(log n).

This is one of the most important tools. Instead of checking every element, you cut the search space in half each time.

The 3 Templates You Need

Template 1: Exact Match

Find the exact value. Classic binary search.

function binarySearch(arr, target) {
  let lo = 0, hi = arr.length - 1;
  while (lo <= hi) {
    let mid = Math.floor((lo + hi) / 2);
    if (arr[mid] === target) return mid;
    if (arr[mid] < target) lo = mid + 1;
    else hi = mid - 1;
  }
  return -1;
}

Template 2: First/Left Position

Find the first occurrence or leftmost position where condition is true.

function lowerBound(arr, target) {
  let lo = 0, hi = arr.length;
  while (lo < hi) {
    let mid = Math.floor((lo + hi) / 2);
    if (arr[mid] < target) lo = mid + 1;
    else hi = mid;  // don't skip — might be the answer
  }
  return lo;
}

Template 3: Search on Answer Space

Not searching the array — searching for the minimum/maximum valid answer.

// Example: "minimum capacity to ship packages in D days"
let lo = maxPackage, hi = totalWeight;
while (lo < hi) {
  let mid = Math.floor((lo + hi) / 2);
  if (canShipInDays(mid, D)) hi = mid;
  else lo = mid + 1;
}
Interview trap: Many "hard" problems are just binary search in disguise. Keywords: "minimum/maximum that satisfies...", "smallest/largest possible..."

2D Arrays (Matrix)

What Is a Matrix?

An array of arrays. Think of it as a grid with rows and columns.

3×4 Matrix — Click a cell
Access: matrix[row][col]

Key Matrix Traversals

Row by Row

for (let r = 0; r < rows; r++)
  for (let c = 0; c < cols; c++)
    visit(matrix[r][c]);

Column by Column

for (let c = 0; c < cols; c++)
  for (let r = 0; r < rows; r++)
    visit(matrix[r][c]);

Diagonal

for (let i = 0; i < n; i++)
  visit(matrix[i][i]); // main diagonal

4 Neighbors (BFS/DFS)

const dirs = [[-1,0],[1,0],[0,-1],[0,1]];
for (let [dr,dc] of dirs) {
  let nr = r+dr, nc = c+dc;
  if (inBounds(nr,nc)) visit(nr,nc);
}

Rotate Matrix 90° — The Classic

The trick: Transpose + Reverse each row

// Step 1: Transpose (swap rows ↔ columns)
for (let i = 0; i < n; i++)
  for (let j = i+1; j < n; j++)
    [matrix[i][j], matrix[j][i]] = [matrix[j][i], matrix[i][j]];

// Step 2: Reverse each row
for (let row of matrix) row.reverse();
Spiral traversal is another classic: Use 4 boundaries (top, bottom, left, right) and shrink them inward as you go around.

Essential Array Tricks

In-Place Operations (O(1) Space)

Reverse an Array

let l = 0, r = arr.length - 1;
while (l < r) {
  [arr[l], arr[r]] = [arr[r], arr[l]];
  l++; r--;
}

Rotate Array by k Steps

// [1,2,3,4,5,6,7] rotate by 3 → [5,6,7,1,2,3,4]
// Trick: 3 reverses!
k = k % arr.length;
reverse(arr, 0, arr.length-1);  // reverse all
reverse(arr, 0, k-1);            // reverse first k
reverse(arr, k, arr.length-1);   // reverse rest

Move Zeros to End

let writePos = 0;
for (let i = 0; i < arr.length; i++) {
  if (arr[i] !== 0) {
    [arr[writePos], arr[i]] = [arr[i], arr[writePos]];
    writePos++;
  }
}

Use Array as HashMap (Index as Key)

When values are in a known range (like 1 to n), use the index itself to mark seen values:

// Find the missing number in [1..n]
function missingNumber(nums) {
  const n = nums.length;
  const expected = n * (n + 1) / 2;  // sum formula
  const actual = nums.reduce((a, b) => a + b, 0);
  return expected - actual;
}
Cycle sort / marking trick: For "find duplicates in [1..n]" — negate the value at index abs(num)-1. If already negative, it's a duplicate!

Boyer-Moore Voting Algorithm

Find the majority element (appears more than n/2 times) in O(n) time, O(1) space:

function majorityElement(nums) {
  let candidate = null, count = 0;

  for (let num of nums) {
    if (count === 0) candidate = num;
    count += (num === candidate) ? 1 : -1;
  }
  return candidate;
}
Intuition

Think of it as a voting battle. Each element "votes" for itself. When two different elements clash, they cancel out. The majority element always wins because it has more votes than all others combined.
