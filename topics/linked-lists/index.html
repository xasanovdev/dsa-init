What Is a Linked List?

The One-Sentence Answer

A linked list is a chain of nodes where each node stores a value and a pointer to the next node. Unlike arrays, nodes can live anywhere in memory.

Real-World Analogy

A treasure hunt. Each clue has a message (the value) and directions to the NEXT clue (the pointer). You can't jump to clue #5 directly — you must follow the chain from clue #1 → #2 → #3 → #4 → #5.

Or think of a train: each car (node) is connected to the next car. To reach car #7, you walk through cars #1 through #6.

Linked lists build the foundation for stacks, queues, hash table chaining, LRU caches, and graph adjacency lists. They test your pointer manipulation skills — a core interview skill.

The Node — Building Block

class ListNode {
  constructor(val) {
    this.val = val;
    this.next = null;  // pointer to next node
  }
}

// Build a linked list: 1 → 2 → 3 → null
const head = new ListNode(1);
head.next = new ListNode(2);
head.next.next = new ListNode(3);
Key concept: You only have access to the head (first node). To reach any other node, you must follow the .next pointers one by one. That's why search is O(n).

Code: Core Operations

// ═══ INSERT AT HEAD — O(1) ═══
function insertAtHead(head, val) {
  const newNode = new ListNode(val);
  newNode.next = head;  // point new node to old head
  return newNode;       // new node IS the new head
}

// ═══ INSERT AT TAIL — O(n) ═══
function insertAtTail(head, val) {
  const newNode = new ListNode(val);
  if (!head) return newNode;
  let curr = head;
  while (curr.next) curr = curr.next; // walk to end
  curr.next = newNode;
  return head;
}

// ═══ DELETE NODE — O(1) if you have previous ═══
function deleteNode(head, target) {
  if (head.val === target) return head.next; // delete head
  let curr = head;
  while (curr.next && curr.next.val !== target)
    curr = curr.next;
  if (curr.next) curr.next = curr.next.next; // skip over
  return head;
}

// ═══ SEARCH — O(n) ═══
function search(head, target) {
  let curr = head;
  while (curr) {
    if (curr.val === target) return curr;
    curr = curr.next;
  }
  return null; // not found
}

The Dummy Head Trick

Many linked list problems become easier with a dummy node before the real head:

const dummy = new ListNode(0);
dummy.next = head;

// Now you never worry about edge cases with the head!
// At the end: return dummy.next (the real head)
Why? Without dummy, deleting the head node is a special case. With dummy, EVERY node (including head) has a "previous" node, so all deletes use the same logic. Use it whenever the head might change.

class DoublyListNode {

    this.val = val;

Doubly linked advantage: Delete a node in O(1) without needing to know the previous node — because each node already has a .prev pointer.

Pattern: Fast & Slow Pointers

What Is It?

Two pointers traverse the list at different speeds. Slow moves 1 step, fast moves 2 steps. When fast reaches the end, slow is at the middle!

Real-World

Two runners on a track. One runs twice as fast. When the fast runner finishes the lap, the slow runner is exactly at the halfway point. No need to measure the track length first!

Code: Find Middle Node

function findMiddle(head) {
  let slow = head;
  let fast = head;

  while (fast && fast.next) {
    slow = slow.next;       // 1 step
    fast = fast.next.next;  // 2 steps
  }

  return slow; // slow is at the middle!
}
Why it works: Fast travels 2× the speed. When fast hits the end (distance n), slow has traveled n/2. One pass, O(1) space, no need to know the length!

Fast & Slow Use Cases

Find Middle

Slow = 1 step, Fast = 2 steps. When fast ends, slow = middle.

Detect Cycle

If there's a cycle, fast and slow WILL eventually meet (Floyd's algorithm).

Find Cycle Start

After they meet, reset one to head. Move both 1 step. They meet at cycle start.

Kth from End

Advance fast by k steps. Then move both 1 step. When fast ends, slow is k from end.

Pattern: Reverse a Linked List

The #1 Linked List Operation

Reversing a linked list is the most important linked list skill. It appears directly in interviews AND as a sub-step in many other problems.

Real-World

Reversing a train. Detach each car and reattach it to the front of a new train. Car by car, the order flips.

Code: Iterative Reverse

function reverseList(head) {
  let prev = null;
  let curr = head;

  while (curr) {
    const next = curr.next;  // save next
    curr.next = prev;        // flip the pointer!
    prev = curr;             // advance prev
    curr = next;             // advance curr
  }

  return prev; // prev is the new head
}
The 3 variables: prev, curr, next. Save the next pointer, flip the current, advance both. That's the entire algorithm!

Code: Recursive Reverse

function reverseListRecursive(head) {
  if (!head || !head.next) return head;

  const newHead = reverseListRecursive(head.next);
  head.next.next = head;  // flip: my next points back to me
  head.next = null;       // I now point to null

  return newHead;
}

Reverse a Sub-section (Reverse Between)

// Reverse nodes from position left to right
function reverseBetween(head, left, right) {
  const dummy = new ListNode(0);
  dummy.next = head;
  let pre = dummy;

  // Move pre to the node BEFORE left
  for (let i = 1; i < left; i++) pre = pre.next;

  // Reverse the sub-section
  let curr = pre.next;
  for (let i = 0; i < right - left; i++) {
    const next = curr.next;
    curr.next = next.next;
    next.next = pre.next;
    pre.next = next;
  }

  return dummy.next;
}
This is a common interview follow-up: "Now reverse only part of the list." Use a dummy node + the same reverse logic on the sub-section.

Pattern: Merge Two Sorted Lists

The Classic Interview Problem

Given two sorted linked lists, merge them into one sorted list. This is LeetCode #21 and a building block for merge sort on linked lists.

Real-World

Two lines of students sorted by height. Compare the front of each line. The shorter person goes first. Repeat until both lines are empty.

Code: Merge Two Sorted Lists

function mergeTwoLists(l1, l2) {
  const dummy = new ListNode(0);
  let tail = dummy;

  while (l1 && l2) {
    if (l1.val <= l2.val) {
      tail.next = l1;
      l1 = l1.next;
    } else {
      tail.next = l2;
      l2 = l2.next;
    }
    tail = tail.next;
  }

  tail.next = l1 || l2; // append remaining
  return dummy.next;
}
Dummy node shines here! We don't know which list has the smaller first element. Dummy node lets us build the result without edge cases.

Pattern: Cycle Detection (Floyd's Algorithm)

The Problem

Does this linked list have a cycle? If the last node points back to some earlier node, you'll loop forever.

Real-World

A race track. If the track is circular, a faster runner will eventually lap the slower runner and they'll meet. If the track is straight, the fast runner just finishes first. Meeting = cycle. No meeting = no cycle.

Code: Detect Cycle + Find Cycle Start

// ═══ DETECT CYCLE ═══
function hasCycle(head) {
  let slow = head, fast = head;

  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) return true;  // they met = cycle!
  }

  return false; // fast reached end = no cycle
}

// ═══ FIND WHERE THE CYCLE STARTS ═══
function detectCycleStart(head) {
  let slow = head, fast = head;

  // Phase 1: Find meeting point
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) break;
  }
  if (!fast || !fast.next) return null; // no cycle

  // Phase 2: Reset slow to head, move both 1 step
  slow = head;
  while (slow !== fast) {
    slow = slow.next;
    fast = fast.next; // now both move 1 step!
  }
  return slow; // they meet at the cycle start
}
Phase 2 math: If the cycle starts at distance d from head, and they met inside the cycle — resetting slow to head and moving both 1 step guarantees they meet at the cycle start. This is Floyd's genius.

More Essential Patterns

Pattern: Remove Nth Node From End

function removeNthFromEnd(head, n) {
  const dummy = new ListNode(0);
  dummy.next = head;
  let fast = dummy, slow = dummy;

  // Advance fast by n+1 steps (gap of n between them)
  for (let i = 0; i <= n; i++) fast = fast.next;

  // Move both until fast reaches end
  while (fast) {
    slow = slow.next;
    fast = fast.next;
  }

  slow.next = slow.next.next; // skip the nth node
  return dummy.next;
}
The trick: Keep a gap of n nodes between slow and fast. When fast reaches the end, slow is exactly at the node BEFORE the one to delete. One pass!

Pattern: Palindrome Linked List

function isPalindrome(head) {
  // Step 1: Find middle (fast & slow)
  let slow = head, fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
  }

  // Step 2: Reverse second half
  let prev = null;
  while (slow) {
    const next = slow.next;
    slow.next = prev;
    prev = slow;
    slow = next;
  }

  // Step 3: Compare both halves
  let left = head, right = prev;
  while (right) {
    if (left.val !== right.val) return false;
    left = left.next;
    right = right.next;
  }
  return true;
}
Combines 2 patterns!

Fast & slow to find middle + reverse second half + compare. This is a great example of how patterns compose together.

Pattern: Reorder List

Reorder L0→L1→...→Ln to L0→Ln→L1→Ln-1→L2→Ln-2→...

function reorderList(head) {
  // 1. Find middle
  let slow = head, fast = head;
  while (fast.next && fast.next.next) {
    slow = slow.next;
    fast = fast.next.next;
  }

  // 2. Reverse second half
  let prev = null, curr = slow.next;
  slow.next = null; // cut the list
  while (curr) {
    const next = curr.next;
    curr.next = prev; prev = curr; curr = next;
  }

  // 3. Merge alternately
  let first = head, second = prev;
  while (second) {
    const tmp1 = first.next, tmp2 = second.next;
    first.next = second;
    second.next = tmp1;
    first = tmp1; second = tmp2;
  }
}
Three patterns in one: Find middle + Reverse + Merge. If you master the individual patterns, "hard" problems become combinations of easy steps.

Pattern: Add Two Numbers

// Numbers stored in reverse: 342 = 2→4→3
function addTwoNumbers(l1, l2) {
  const dummy = new ListNode(0);
  let curr = dummy, carry = 0;

  while (l1 || l2 || carry) {
    const sum = (l1?.val || 0) + (l2?.val || 0) + carry;
    carry = Math.floor(sum / 10);
    curr.next = new ListNode(sum % 10);
    curr = curr.next;
    l1 = l1?.next;
    l2 = l2?.next;
  }

  return dummy.next;
}

Pattern: LRU Cache (Doubly Linked List + HashMap)

class LRUCache {
  constructor(capacity) {
    this.cap = capacity;
    this.map = new Map();
    // Dummy head and tail for easy insert/remove
    this.head = new DoublyListNode(0);
    this.tail = new DoublyListNode(0);
    this.head.next = this.tail;
    this.tail.prev = this.head;
  }

  get(key) {
    if (!this.map.has(key)) return -1;
    const node = this.map.get(key);
    this._remove(node);
    this._addToFront(node);
    return node.val;
  }

  put(key, value) {
    if (this.map.has(key)) this._remove(this.map.get(key));
    const node = new DoublyListNode(key, value);
    this._addToFront(node);
    this.map.set(key, node);
    if (this.map.size > this.cap) {
      const lru = this.tail.prev; // least recently used
      this._remove(lru);
      this.map.delete(lru.key);
    }
  }
}
LRU Cache = HashMap + Doubly Linked List. Map gives O(1) lookup. DLL gives O(1) insert/remove at any position. Most recently used at front, least recently used at back. This is a VERY common interview question (LeetCode #146).

Essential Tricks & Techniques

Trick 1: Sentinel / Dummy Node

Create a fake node before the head to simplify edge cases:

const dummy = new ListNode(0);
dummy.next = head;
// ... do your work ...
return dummy.next; // real head

Use when: Head might change (delete head, insert before head, merge two lists).

Trick 2: Multiple Passes Are OK

Don't be afraid to traverse the list more than once:

  • Pass 1: Find the length
  • Pass 2: Use the length to find the target node

Two O(n) passes = still O(n). Don't over-optimize when clarity matters.

Trick 3: Draw It Out!

The #1 debugging technique

Linked list bugs are almost always pointer errors. Draw boxes and arrows on paper. Trace through each step. Check: did I lose a reference? Am I updating in the right order?

Common mistakes: (1) Losing reference to next before saving it. (2) Creating a cycle by accident. (3) Forgetting to handle null/empty.

Trick 4: In-place vs Extra Space

In-place (O(1) space)

Modify pointers directly. Used for reverse, reorder, partition. Harder but more impressive.

Extra space (O(n))

Copy to array, solve there, copy back. Simpler but uses more memory. Good for first attempt.

Trick 5: Sort a Linked List (Merge Sort)

function sortList(head) {
  if (!head || !head.next) return head;

  // Split in half (fast & slow)
  let slow = head, fast = head.next;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
  }
  const mid = slow.next;
  slow.next = null; // cut!

  // Sort each half
  const left = sortList(head);
  const right = sortList(mid);

  // Merge sorted halves
  return mergeTwoLists(left, right);
}
Why merge sort? Quick sort needs random access (bad for linked lists). Merge sort works perfectly: split at middle (fast & slow), sort halves, merge. O(n log n) time, O(log n) stack space.
