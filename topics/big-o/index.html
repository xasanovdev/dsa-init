What Is Big O & Why Care?

The One-Sentence Answer

Big O tells you: "As my input gets bigger, how much MORE work does my code do?"

It doesn't measure exact seconds. It measures the growth rate — how the work scales.

The Story That Explains Everything

Real-World Analogy

Imagine a phone book with everyone's name and number.

Your friend asks: "What is Alice's phone number?"

Method 1 (Slow): Read every single name from page 1 until you find "Alice." If 1,000,000 names, maybe 1,000,000 checks. That's O(n).

Method 2 (Fast): Open the middle. Is "Alice" before or after? Throw away half. Repeat. About 20 checks for 1,000,000 names. That's O(log n).

Same problem, wildly different speeds. Big O helps you pick the right method.

O(1) — Constant Time

What Does It Mean?

No matter how big the input, the work stays the same. 10 items or 10 billion — same steps.

Real-World

Checking your mailbox. Doesn't matter how many letters exist in the world. You walk to YOUR box and open it. Same work every time.

Turning on a light switch. Room size doesn't matter. One flip = done.

Why "log"? The Tree

Each halving is one level of a tree. 8 items = 3 levels deep = log₂(8) = 3 steps max.

8 → 4 → 2 → 1 = 3 steps = log₂(8)
Spot it: Problem gets halved each step. Keywords: "sorted", "binary search", "divide".

O(n) — Linear Time

What Does It Mean?

You look at every item once. 100 items = ~100 steps. 1 million = ~1 million. Fair and proportional.

Real-World

Reading a book. 300 pages = read 300 pages. Twice as many pages = twice the work.

Counting people in a line. Point at each person: "1, 2, 3..." More people = more counting.

Why O(2n) = O(n)?

Big O drops constants. It only cares about the shape of growth.

O(2n) → drop the 2 → O(n)
O(5n + 3) → drop 5 and 3 → O(n)
O(n/2) → drop the /2 → O(n)
O(100) → constant → O(1)
Spot it: A single loop going through the whole input. Two separate (non-nested) loops = still O(n).
n·log

O(n log n) — Linearithmic

What Does It Mean?

The best possible sorting speed (for comparison sorts). Split into halves (log n levels), do n work at each level.

Real-World

Sorting cards with Merge Sort. Split the deck, sort each half, merge back. Splitting goes log n deep, merging touches all n cards each level.

O(n²) — Quadratic Time

What Does It Mean?

For each item, you check every OTHER item. Loop inside a loop. 100 items = 10,000 steps!

Real-World

Handshakes at a party. Every person shakes hands with every other person. 10 people ≈ 45 handshakes. 100 people ≈ 4,950. 1000 people ≈ 500,000!

O(2ⁿ) — Exponential Time

What Does It Mean?

Work DOUBLES with each additional item. 10 items = 1,024 steps. 20 items = over 1 MILLION. 30 items = over 1 BILLION.

Real-World

Rice on a chessboard. 1 grain on square 1, 2 on square 2, 4 on square 3... By square 64, more rice than exists on Earth!

Outfit choices. For each clothing item: "wear it" or "don't." 20 items = 2²⁰ = 1,048,576 outfits.

n!

O(n!) — Factorial Time

What Does It Mean?

The slowest thing you'll ever see. n! means n × (n-1) × (n-2) × ... × 1. It grows so fast your brain can't even imagine it.

5! = 5 × 4 × 3 × 2 × 1 = 120

10! = 3,628,800 (over 3 million)

20! = 2,432,902,008,176,640,000 (2.4 QUINTILLION)

Real-World

Arranging people in a line. How many different orders can 5 people stand in? The first spot has 5 choices. The second spot has 4. Then 3, 2, 1. Total: 5 × 4 × 3 × 2 × 1 = 120 arrangements.

With 10 people? Over 3.6 million arrangements.

With 20 people? More arrangements than atoms in your body.

Code Example — The Traveling Salesman (Brute Force)

// Visit every city exactly once. What's the shortest route?
// Brute force: try ALL possible orderings = O(n!)

function permute(cities, path = []) {
  if (cities.length === 0) {
    checkDistance(path);  // is this the shortest?
    return;
  }
  for (let i = 0; i < cities.length; i++) {
    const remaining = [...cities];
    const city = remaining.splice(i, 1);
    permute(remaining, [...path, city]);
  }
}
// 10 cities = 3,628,800 routes to check
// 15 cities = 1,307,674,368,000 routes (IMPOSSIBLE)
When you see O(n!): This is almost always "generate all permutations." Classic problems: Traveling Salesman (brute force), N-Queens (worst case). In interviews, you need to find a smarter approach!
O(n!) vs O(2ⁿ): Factorial is even WORSE than exponential! At n=15, 2¹⁵ = 32,768 but 15! = 1,307,674,368,000. That's 40 million times worse!

Space Complexity

Same Idea, Different Resource

Time = how much time. Space = how much memory.

Real-World

Cooking. Time complexity = how long you cook. Space complexity = how many pots and pans you use.

SpaceMeaningExample
O(1)No extra memorySwap two variables
O(n)Grows with inputCopy an array, hash map
O(n²)2D tableDP grid for two strings
Interview tip: Always mention BOTH. "This is O(n) time and O(1) space."

The 7 Rules for Calculating Big O

Rule 1: Drop Constants
O(2n) → O(n)  |  O(500) → O(1)
Rule 2: Drop Smaller Terms
O(n² + n) → O(n²)  |  O(n + log n) → O(n)
Rule 3: One Loop = O(n)
A single for loop through n items
Rule 4: Nested Loops = Multiply
Loop inside a loop → O(n × n) = O(n²)
Rule 5: Sequential Loops = Add
Loop then another loop (not nested) = O(n + n) = O(n)
Rule 6: Halving = O(log n)
Divide the data in half each step
Rule 7: Different Inputs = Different Variables
Loop A (size n) then B (size m) → O(n + m)
